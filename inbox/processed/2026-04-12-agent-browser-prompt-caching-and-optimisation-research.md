## New Principles / Learnings
- **Ctrl→Cmd remapping on macOS**: Computer-use API sends `ctrl+` shortcuts but macOS interprets Control differently from Cmd. Fixed via `normaliseKey` in `actions.ts` remapping `ctrl`/`control` to `Meta` (Cmd). Was silently causing failures in earlier workflows.
  - Discovered in: Schema workflow testing
  - Likely applies to: Any browser agent running on macOS
  - Evidence: Select-all, copy/paste shortcuts failing intermittently; agent recovering but burning iterations
- **Use Ctrl+A not triple-click for textarea selection**: Triple-click only selects one line/paragraph. Ctrl+A (Cmd+A via remap) selects all content in multi-line fields.
  - Discovered in: JSON-LD field interaction
  - Likely applies to: Any workflow touching multi-line input fields
- **Click outside textarea before scrolling**: If cursor is inside a textarea, scroll events scroll the field content, not the modal/page.
  - Discovered in: JSON-LD field in SEO Settings modal
  - Likely applies to: Any workflow with scrollable textareas
- **Save button position varies with textarea content**: Fixed-height fields (title, meta) allow hardcoded Save coordinates. Variable-height fields (JSON-LD) push Save to different positions — needs one screenshot to locate.
  - Discovered in: Schema workflow
  - Likely applies to: Any workflow with variable-height form fields
- **Always check for existing schema before adding**: CMS platforms (Shopify, WordPress, Wix) often auto-generate schema via themes. Adding duplicates can confuse search engines.
  - Discovered in: Schema workflow — Product and FAQ schema already present
  - Likely applies to: Any structured data workflow
- **Trust the agent, not the DOM**: Standalone Playwright scripts with Shadow DOM traversal failed after 2 hours. The visual browser agent handled the same task in 14 iterations for $0.34. Pixel-based agents bypass Shadow DOM, React state, cross-origin iframes entirely.
  - Discovered in: Schema workflow — first Claude Code attempt vs agent run
  - Likely applies to: Any decision about building custom scripts vs using the agent
- **Always verify customer URL in state**: Agent picked up stale customer URL from previous run's state file. New customer runs need explicit state reset.
  - Discovered in: Schema workflow setup
  - Likely applies to: All workflow runs
