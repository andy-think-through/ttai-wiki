# Naming Agents Lands Well
> Giving AI agents human names creates shared vocabulary and makes abstract automation memorable and concrete.
> Last updated: 2026-04-30

## The Principle
When you name an AI agent, clients adopt the name naturally and start talking about the agent as if it were a team member. This creates shared vocabulary, makes the capability memorable, and reduces the "what does it actually do?" friction in follow-up conversations.

## Where Discovered
**AccXel audit (28 Apr 2026):** Charlotte and Natalie referred to "Bob" and "Mark" naturally throughout the afternoon session. They chose the name Bob themselves for the apprenticeship service agent. The naming created immediate shared vocabulary for what would otherwise be "the automated compliance checker."

**Previously:** KW Bell named their invoice OCR agent "Belle" -- same pattern. The name stuck and became how the project was discussed internally.

## Evidence
- AccXel: Bob (apprenticeship agent), Mark (BDM), Sarah (marketing) -- all adopted naturally
- KW Bell: Belle (invoice OCR agent) -- name used in all correspondence
- TTAI internal: Wiki, Fred, Karen, Cord -- employees referred to by name in all Slack reports

## Where It Applies
- All [[mark]] and bespoke agent builds -- name the agent during the pitch
- [[consulting]] audits -- introduce agent names when scoping opportunities
- [[agent-browser]] deployments -- name is part of the deliverable
